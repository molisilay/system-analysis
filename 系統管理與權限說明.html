未完成規劃與待確認
🔨 尚未實作的規劃功能
🔐 忘記密碼 / 自助重設 尚未實作
需提供 Email 驗證或手機 OTP 讓使用者自助重設密碼，目前忘記密碼只能請管理員重新設定。
🛡️ 密碼強度規則 待確認
建議規範最短長度、大小寫、數字、特殊字元，並提供前端即時驗證提示。需確認目前實作。
📦 倉管角色 尚未實作
M5 備品/庫存模組上線前必須補，需要「可看哪幾個倉」的 ABAC 屬性。
🛒 採購角色 尚未實作
M6 採購模組上線前必須補。採購核准須搭配簽核引擎。
🔧 維修工程師角色 尚未實作
目前維修工單由 Admin 兼任指派，規劃中給維修工程師更細緻權限（接單、換件、領料）。
🏷️ ABAC 屬性權限 尚未實作
角色決定「能做什麼操作」，屬性決定「能看哪些資料」。例：倉管 A 只能看倉庫 1、倉管 B 只能看倉庫 2。
📝 簽核引擎 尚未實作
M6 採購核准流程需要：串聯（A→B→C）、條件式（金額分層）、並聯（多人同核）、代理人機制、催辦。
🚫 退簽 / 撤回 / 加簽 尚未實作
搭配簽核引擎的進階功能，讓核准流程更有彈性。
📜 完整稽核軌跡 尚未實作
目前只有刪除稽核，其他操作（新增、編輯、指派、核准）尚無完整稽核日誌。
⚠️ 以下事項需與庭程 / 資訊單位確認
  • 目前密碼規則有哪些（長度、複雜度）？前端輸入時有無即時提示？
  • 管理員建立的新帳號，使用者第一次登入是否強制改密？
  • 一個使用者能不能同時具備多個角色（例：A 既是倉管也是維修工）？
  • 角色權限顆粒度：目前是「模組層級」還是「CRUD 動作層級」？能否只給「讀取」不給「修改」？
  • 有使用者在用的角色，要刪除時系統會擋嗎？還是直接刪除讓使用者失去權限？
  • 公司、場域支援 Excel 批量匯入嗎？（設備已知支援）
  • Super Admin 可否跨場域調派人員？例如把五股的員工暫時調到台南支援？
  • 稽核日誌目前只記「刪除」，其他動作（新增、修改、指派、登入）有沒有保留紀錄？
  • 從稽核日誌能不能還原誤刪的資料？還原需要什麼權限？
  • JWT Token 的有效期多久？Refresh Token 機制是什麼？是否支援強制登出所有裝置？
